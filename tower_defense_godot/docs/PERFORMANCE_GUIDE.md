# 效能指南

本文整理目前 Godot Web 版與本地版的效能規則。後續新增畫面精細度、塔造型、子彈、敵人種類與場景時，應先檢查是否符合這些限制。

## 已知瓶頸

塔防後期主要瓶頸不是單一運算，而是大量物件同時存在：

- 塔很多。
- 敵人很多。
- 子彈很多。
- 衝擊波與飄字很多。
- 透明繪製與大範圍圓形很多。
- UI 每幀重刷。

Web 版尤其要小心 Canvas / WebGL 繪圖成本、中文字型載入、`.wasm` 與 `.pck` 首次下載。

## 現行優化

### 分層繪製

靜態層處理：

- 背景。
- 格線。
- 建造區。
- 塔地板。
- 敵人路徑。
- 部分塔靜態外觀。

動態層處理：

- 敵人。
- 子彈。
- 命中特效。
- 飄字。
- 會旋轉的砲管或發射臂。
- 選取塔範圍。
- 建造預覽。

原則：不會每幀變動的東西，不要放在主 `_draw()` 每幀重畫。

### 高負載低畫質模式

目前依物件量與遊戲速度自動降低繪製：

- 高負載時不繪製子彈，但傷害照算。
- 高負載時一般敵人血條可隱藏，Boss 與審查員保留。
- 高負載時減少衝擊波。
- 忙碌狀態下選取塔範圍只畫外框。
- 速度 `4x`、`8x`、`16x` 時降低子彈與血條繪製。

### UI 更新節流

UI 透過狀態簽名比對，資料未改變時不重刷 Label/Button。

### Hover 檢查節流

建造預覽只有在滑鼠格子改變時才重新檢查，不在同一格每幀重跑路徑搜尋。

### 字型瘦身

`Noto Sans TC` 已縮成遊戲用字子集，降低首次載入成本。

### Netlify 資源快取

核心資源使用 `td-<hash>.*` 檔名，搭配長快取與 gzip/brotli 壓縮。

## 新增功能效能規則

### 新增塔外觀

優先判斷哪些部分是靜態：

- 底座通常是靜態。
- 等級標記通常是靜態，升級時才重畫。
- 砲管、旋轉臂、選取範圍才是動態。

不要每幀重畫整座塔的所有裝飾。

### 新增子彈

必須支援：

- 高負載不繪製，但傷害仍計算。
- 簡化繪製版本。
- 數量很多時抽樣或關閉。

不要讓子彈本身產生大量透明圓形或複雜弧線。

### 新增敵人外觀

必須支援：

- 一般繪製。
- 高負載簡化繪製。
- 方向旋轉。
- Boss 放大。

大量小敵人不要使用太多分段曲線與透明圖層。

### 新增命中特效

必須有：

- 同時存在數量上限。
- 高負載抽樣。
- 生命週期短。
- 可依塔種換色，但不要每次建立複雜資源。

### 新增路徑或地形效果

優先放靜態層。只有路徑變化、建塔、拆塔、顯示路徑開關改變時才重繪。

### 新增 UI

避免每幀更新文字。UI 文字只有數值改變時才更新。

## 效能檢查場景

每次改視覺或戰鬥，至少檢查：

1. 第 1 波：確保基本畫面正常。
2. 第 20 波後：檢查新增敵人與中期負載。
3. 第 40 波後：檢查大量敵人與塔。
4. 第 50 波：檢查 Boss 與攻擊量審查員。
5. 速度 `1x`、`4x`、`16x`：確認降級規則生效。
6. 顯示路徑開關：確認靜態層重繪沒有異常。

## Chrome / Safari 注意

目前 Chrome 表現較穩。Safari 可能對 Godot Web 與大量繪製更敏感。

如果 Safari 卡頓，優先：

- 降低透明特效。
- 降低大面積圓形繪製。
- 減少每幀文字繪製。
- 確認字型已瘦身。
- 確認高負載模式在 Safari 也會啟動。

## 載入速度檢查

Web 版載入慢時，優先看：

- `fonts/NotoSansTC-Regular.ttf` 是否被換回完整字型。
- `.wasm`、`.pck` 是否有 gzip/brotli。
- `index.html` 是否正確指向 `td-<hash>.*`。
- Netlify headers 是否正確。
- 是否把 `builds/**` 打包回 `.pck`。

## 不建議做法

- 每幀完整重畫背景格線。
- 每顆子彈都用多層透明效果。
- 衝擊波沒上限。
- 所有敵人永遠顯示血條。
- 高速模式仍完整顯示所有特效。
- UI label 每幀重設文字。
- 新增大字型或未壓縮圖片後直接部署。

## 後續可做的優化

- 將更多塔外觀拆成靜態與動態兩段。
- 敵人外觀依距離或數量切換細節層級。
- 衝擊波改成更便宜的幾何繪製。
- 對 Web 版加入更明確的畫質選項。
- 建立簡單效能測試關卡，用固定物件量比較改動前後。
